mepTracker¶

  • Documentation: https://yelrew.github.io/m_e_p_t_r_a_c_k_e_r
  • Source: https://github.com/yelrew/meptracker
  • Bug reports: https://github.com/yelrew/meptracker/issues

mepTracker is a package for tracking and finding optimal Motor-Evoked Potential (MEPs). By starting with an initial guess, a path to the region of maximum MEP is determined by reading the coil responses along an optimized path in a 2D or 3D space.

Currently, an interface to the Simplex vertex-edge optimization method (sve) is available. The sve package is required. See the SVE page for more information.

Install¶

The latest version of mepTracker can be installed using pip:

$ pip install --user -U git+ssh://git@github.com/yelrew/meptracker

or directly from the sources:

$ git clone https://github.com/yelrew/meptracker.git && cd meptracker && pip install -e .

Usage¶

The basic mepTracker usage is for determining a 3D path to the region of maximum MEP amplitude by probing the electromyography at optimal stimulation positions. The scanner function (scanfunc) should return the MEP value probed at the specific coordinates passed as input arguments each iteration.

In this example, scanfunc is a 5th-degree polynomial in three variables previously fitted from an offline experiment. The optimal stimulation path is found using the sve method.

>>> import meptracker
>>> import numpy as np
>>> from meptracker.tests.test_tracker_offline_scanfunc import scanfunc # External scanner function
>>>
>>> # Initial guess and parameters for the sve optimizer
>>> x0 = [-50., 0., 100.] # initial guess (average x,y,z from fitted data, in mm)
>>> rMin = 0.0001
>>> rIni = 1
>>> rMax = 116.
>>> stop = 10*rMin
>>>
>>> # Create the tracker object
>>> tracker = meptracker.Tracker(scanfunc, dim=3)
>>>
>>> # Find the optimal x (the tolerance is the maximum function mismatch)
>>> tracker.track(x0, method='sve', direction='POS',
>>>               rIni=rIni, rMin=rMin, rMax=rMax, stop=stop,
>>>               stop_if_dFx_lt_tol=True, tol_dfx = .1)
>>>
>>> X, Fx, v, nit, nev, maxit = tracker.results.values()
>>>
>>> np.set_printoptions(suppress=True)
>>> print('\n**********')
>>> print('\ntest_tracker_offline\n')
>>> print("Perform a 3D MEP optimization considering the scanner function ", end='')
>>> print("as a fitting function obtained from offline MEP data.")
>>> print("\nProblem dimension: ", X.shape[1])
>>> print("Method used: sve\n")
>>> print("Simplex probe points: ", v.shape[1])
>>> print("\nResults:")
>>> print('\nConvergence in', nit, 'iterations\n')
>>> print('Summary of the iterative process\n')
>>> print(nit, 'iterations')
>>> print(nev, 'function evaluations')
>>> print()
>>> n = X.shape[1]
>>> print('{:^5}{:^12}{:^12}{:^12}{:^12}'.format('it', 'x1', 'x2', 'x3', 'f(x*)'))
>>> fmtr = '{:^5}' + '{:^12.5}'*(n+1)
>>> [print(fmtr.format(i, X[i,0], X[i,1], X[i,2], Fx[i])) for i in [*range(nit+1)]]
>>> print()

If map2D is True, mepTracker will map the 3D input into a 2D polar projection. The bi-dimensional optimizer output will be converted back to 3D by assuming a fixed radius from the original guess. See the test_tracker_offline_2D test case for more details.
